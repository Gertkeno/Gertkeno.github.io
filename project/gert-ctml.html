A Simple Html Converter

My inspiration for this project was HAML, I liked the automatic end tags and generally cleaner looking code.

Unfortunately all the ruby related features didn't appeal to me, and simply adding odd attributes was harder than just using HTML.

I just wanted to make static websites easier to code. So I started with redefining tags.

From <p> to [p I figured it would make more sense to keep the brackets open if the tag is open. With simplicity and container syntax in mind all end tags went from </p> to a global ].

There are some cases where you don't want to make an end tag, such as <br> and <hr>. For thos cases you just append ]), now with context: [br])

As an example my first paragraph converts like so:

[p My inspiration for this project was [a@http://haml.info/ HAML], I liked the automatic end tags and generally cleaner looking code.]
<p>My inspiration for this project was <a href="http://haml.info/">HAML</a>, I liked the automatic end tags and generally cleaner looking code.</p>

What happened with the [a@... tag though? I guess this requires a expertly crafted transitional question.


Quick Attributes

With tags in place I needed a way to add attributes, starting with the obvious classes and ids I copied straight from HAML. If you added a dot or octothorpe it will append the specified class or id attribute.

Take the previous example as an example.

[div.code [p My inspiration for this project was [a@http://haml.info/ HAML], I liked the automatic end tags and generally cleaner looking code.] ]
<div class="code"> <p>My inspiration for this project was <a href="http://haml.info/">HAML</a>, I liked the automatic end tags and generally cleaner looking code.</p> </div>

When the CSS staples were out of the way I went to actually test my mess of code it's all one file save me.

I found that I couldn't add links, or image sources, the possibilities were stunted. As far as I know HAML didn't have a quick href/src solution so I had to make up my own.

Without straying far from the single special character syntax, I decided on @ and $ since @ makes me think of emails and that's kind of href related. $ looks like s, so $rc.

Initially this didn't work, as I used the same code from classes and ids which only parsed alphabetical characters

I'll dial back the recursion for this example, with this broken code the first paragraph's [a@ tag would look like this

[a@http://haml.info/ HAML]
<a href="http" class="info">HAML</a>

Luckily this meant I could link to a local file named 'http' so it wasn't completely useless.

Besides adding more syntax I decided the best way to fix this was to make href and src notation parse until white space. A space was already required for the basic tags to work, the only difference now is the href or src attribute has to be at the end of tag declaration.

Class + href doesn't come up a lot, and I'm pretty sure most people don't recommend classed anchor tags. That means I had to make my own example of how this syntax works.

[img.pepsi-dog$http://i.imgur.com/BKCGCeK.jpg ])
<img class="pepsi-dog" src="http://i.imgur.com/BKCGCeK.jpg">

Now I know what you've been thinking, probably from the start "All these attribute shortcuts are fantastic, but you can't seriously have one for every attribute! There are only 26 special characters left on my keyboard!"

You're right, and I don't, However I have included a way to add any attribute you want.

[meta{charset="utf-8"}])
<meta charset="utf-8">

Anything inside the curly brackets is inserted plain text into the tag. You can still mix and match quick attributes with this so it doesn't hinder the system I strive to build.

That's probably more than enough about these five shortcuts and their interactions, so I'll move on to less useful, more back-end stuff.


Comments

Pretty basic stuff for this, I just check for a * and append the usual HTML comment syntax <!-- and then the rest of the line -->

I initially didn't include the comments in the final HTML file since HTML comments could be written directly. It's now a command line argument, if you want gert-ctml comments removed include -s when running

Sadly I don't care about multi-line comments in HTML, nor do I care about parsing comments so it's contents is in HTML.

*[p this is sample text, please ignore me]
<!--[p this is sample text, please ignore me]-->

I do however, care about multiple comments on one line. Adding another * to a commented line will stop the comment, or start commenting again.

*this*is*going*to*be*a*wild*ride*my*dudes
<!--this-->is<!--going-->to<!--be-->a<!--wild-->ride<!--my-->dudes

This document, and it's examples couldn't be possible without HTML symbol entities. However I still decided to include character escapes by \

It's like most any character escape, ignores the first \ and prints the second character.
